# CEWPOS Agency-Free Fabric — UX-Knob + Capability Surface Catalog

> **Framing.** Every knob below is a *config object* — and in CEWPOS a config object **is** a CEG attestation (`config:v1` / `scores` row), self-signed at **node-tier** (`ConfigScope::Local`) or **owner-self-tier** (`ConfigScope::Identity`), append-only, latest-wins, version-chained, and hierarchically contained (node ⊂ owner-self ⊂ community ⊂ federation ⊂ constitutional). Because each knob is an attestation row, **each knob is also a renderable 3D view-form**: a signed object with a tier (depth), a capability (color/cohort), a layer (fabric=cell-membrane / boundary=door / agent=brain), and a precedence/version lineage (supersedes chain). This catalog is the **concrete knob/policy/capability surface** the repos expose — "everything but the agent." The abstract object/transformation grammar lives in the separate Constitution map; this is the dial board.
>
> **Layer legend:** `fabric` = mechanism inside the cell membrane · `boundary` = a door/gate (admission, refusal, kill-switch) · `agent` = brain-side composition on top of the fabric.
> **Tier legend:** `node` < `owner-self` < `community` < `federation` < `constitutional` (containment hierarchy; `both` = the Local/Identity config axis itself).
>
> **Total distinct knob entries: 982** across 8 repos (Persist 109 · Verify 93 · Edge 133 · NodeCore 180 · Registry 171 · Server 101 · Agent 98 · Lens 97). Tables below dedupe and group obvious config families.
